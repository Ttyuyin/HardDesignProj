## 四、关键发现

### 4.1 编码检测

1. **BOM 锚点检测 100% 可靠** — UTF-8 (EF BB BF)、UTF-16 LE (FF FE)、UTF-16 BE (FE FF) 均精确识别，confidence=1.0
2. **纯 ASCII 短路有效** — ASCII 文件直接返回，不进入 CJK Agent 分析，避免误判
3. **CJK 编码重叠问题** — GBK/Big5/Shift-JIS 字节范围大面积重叠，< 50 字节样本无法可靠区分
4. **内容裁决成功案例** — `07_shiftjis_japanese.txt` 在 GBK 与 Shift-JIS 平局 (差距 0.4%) 时，通过 kana ratio 检测正确裁决为 Shift-JIS

### 4.2 编码转换

1. **转换引擎稳定性** — 235 次转换 (135+100) **零崩溃**
2. **GB2312 字符集局限** — 唯一出现验证失败的编码，因其字符数最少 (6763)
3. **检测决定转换质量** — 源编码检测错误导致 28 次矩阵测试中出现 0% 匹配
4. **GBK 源编码最通用** — 在所有文本类型中检测准确率最高 (81.2%)

### 4.3 软件功能

1. **UI 组件构建正确** — MainWindow/两个标签页/表格/按钮全部正常实例化
2. **全链路可用** — 文件选择 → 编码检测 → 字符分析 → 编码转换 → 验证显示 流程完整
3. **输出文件合规** — 所有转换输出文件命名规范、内容可解码、非空

## 五、局限与改进建议

| 编号 | 局限 | 影响 | 改进建议 |
|:---:|------|------|---------|
| 1 | CJK 编码在小样本 (< 100B) 下区分困难 | 2/15 检测失败 | 增加样本长度检测，< 100B 时给出低置信度警告 |
| 2 | GB2312 转换验证失败 | 5/135 验证未通过 | 在 UI 中增加字符集范围提示，告知用户 GB2312 的局限 |
| 3 | UTF-16LE BOM-less 非英文文本检测不稳定 | 矩阵测试中多次检测偏差 | 增强 UTF-16 null 字节分析算法 |
| 4 | 无集成测试/CI | 回归测试需手动执行 | 增加 GitHub Actions / pytest 集成 |
